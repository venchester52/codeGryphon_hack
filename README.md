# MVP Marketing AI Auditor

Универсальный Streamlit-инструмент для быстрого маркетингового аудита по табличным данным.

Приложение:
- поддерживает загрузку **CSV, Excel, JSON, SQL и SQLite**;
- приводит любой поддерживаемый источник к единому `pandas.DataFrame` перед анализом;
- автоматически сопоставляет колонки со внутренней схемой;
- динамически считает только доступные метрики;
- показывает слабые объявления и практичные рекомендации;
- дает ответы AI-ассистента на основе текущего контекста кампании;
- сохраняет историю анализов и чатов в SQLite;
- поддерживает регистрацию, вход и выход по email + пароль;
- умеет строить **опциональные дашборды по запросу пользователя** (не включены по умолчанию).

---

## Обновленный интерфейс (UX)

Интерфейс перестроен в формат аналитического рабочего пространства с фокусом на скорость получения инсайтов:

- **Боковая панель (sidebar)** содержит глобальные действия: загрузка источника, параметры источника, открытие сохраненной сессии, настройки дашбордов, профиль и выход.
- **Summary-first сценарий**: после анализа сначала показываются KPI, сигналы риска, сильные/слабые объявления и короткое резюме.
- **Основная область разбита на вкладки**: `Overview`, `Data Mapping`, `Metrics & Results`, `Dashboards`, `Recommendations`, `AI Assistant`, `History`.
- **Дашборды остаются опциональными** и включаются только по явному действию в sidebar.
- **AI Assistant вынесен в отдельную осмысленную зону**, работает на основе текущего контекста анализа.
- **Чистая визуальная иерархия**: меньше перегруженных вертикальных блоков, больше группировки и деталей по требованию.

## Настройка `.env`

1. Скопируйте `.env.example` в `.env`.
2. Заполните переменные для GigaChat.
3. При необходимости настройте путь к SQLite.

Пример:

```env
GIGACHAT_CREDENTIALS=your_gigachat_authorization_key_here
GIGACHAT_SCOPE=GIGACHAT_API_PERS
GIGACHAT_CA_BUNDLE_FILE=/absolute/path/to/russian_trusted_root_ca_pem.crt

# Опционально: явный путь к sqlite-файлу
SQLITE_DB_PATH=data/marketing_auditor.sqlite3

# Опционально: полный SQLAlchemy URL (приоритетнее SQLITE_DB_PATH)
DATABASE_URL=
```

## Поддерживаемые форматы входных данных

### 1) CSV
- Используется `pandas.read_csv`.
- Поведение полностью совместимо с прежним CSV-потоком.

### 2) Excel (`.xlsx`, `.xls`)
- Используется `pandas.read_excel`.
- Если лист один — выбирается автоматически.
- Если листов несколько:
  - приложение пытается выбрать лучший лист автоматически (по непустым строкам и колонкам);
  - при неоднозначности или наличии нескольких релевантных листов пользователь выбирает лист в UI.

### 3) JSON (`.json`)
- Поддерживаются типовые структуры:
  - массив объектов (`list of records`);
  - вложенные объекты, содержащие массив записей;
  - массивы простых значений (как fallback, в колонку `value`).
- Используется нормализация через `pandas.json_normalize`.
- Если найдено несколько путей к записям, в UI показывается выбор JSON-пути.
- Если JSON нельзя привести к полезной таблице, показывается понятная ошибка.

### 4) SQL (`.sql`)
Поддерживаются практичные сценарии для хакатона:
- SQL-дамп с `CREATE TABLE` + `INSERT` (через временную in-memory SQLite и `executescript`);
- SQL-файл только с `INSERT` (через встроенный парсер INSERT);
- SQL-файл только с `SELECT`:
  - анализ невозможен без источника данных,
  - приложение показывает понятное объяснение в UI.

### 5) SQLite (`.sqlite`, `.sqlite3`, `.db`)
- Если загружен SQLite-файл, приложение читает список таблиц.
- Пользователь может выбрать таблицу для анализа, если таблиц несколько.

## Универсальный ingestion-слой

Добавлен модуль:

```text
utils/file_loader.py
```

Он:
- определяет тип источника по расширению;
- подготавливает опции выбора (лист Excel, путь JSON, таблица SQL/SQLite);
- загружает данные в единый `pandas.DataFrame`;
- возвращает метаданные источника (тип, метод парсинга, выбранный лист/путь/таблица, размер данных);
- выбрасывает понятные ошибки для UI при невозможности парсинга.

Все дальнейшие этапы анализа используют тот же конвейер, что и раньше:
- schema mapping;
- валидация обязательных полей;
- расчет метрик;
- анализ слабых объявлений;
- рекомендации;
- формирование `campaign_context`;
- AI-чат и сохранение в БД.

## Опциональные дашборды (по запросу пользователя)

Дашборды **не отображаются автоматически** после загрузки файла и анализа.

По умолчанию пользователь видит основной легкий поток:
- сводные KPI;
- обнаруженные метрики;
- таблицу результатов;
- короткие рекомендации;
- AI-ассистент.

### Как включить дашборды

1. Выполните анализ файла (как обычно).
2. В боковой панели включите чекбокс **«Включить дашборды»**.
3. Выберите группировку, метрику, тип графика, top-N и фильтр по слабым объявлениям.
4. Откройте вкладку **`Dashboards`**.

Пока чекбокс выключен, графики не строятся.

### Поддерживаемые графики

Графики строятся динамически и только при наличии данных:
- топ по расходу (bar);
- топ по целевым действиям (bar), если доступно поле целевых действий;
- самые слабые по CPA / CPL / CAC (bar), если есть соответствующая метрика;
- самые сильные по ROAS / CTR (bar), если есть соответствующая метрика;
- scatter: расход vs целевые действия;
- scatter: клики vs целевые действия;
- распределение одной из метрик CPA / CPC / CTR (гистограмма через bar);
- распределение слабых / неслабых объявлений (bar);
- пользовательский график с выбором метрики и типа (bar/line/area).

### Адаптивность и обработка ошибок

- Если нужная метрика или колонка отсутствует — соответствующий график пропускается с понятным сообщением.
- Если после фильтрации/группировки недостаточно данных — выводится дружелюбное уведомление.
- Ошибки в визуализациях не должны ломать приложение: блок дашбордов изолирован безопасной обработкой ошибок.

## Что показывается в UI по источнику

После успешной загрузки отображается блок **«Источник данных»**:
- тип источника;
- метод парсинга;
- выбранный лист/JSON-путь/SQL-режим/таблица (если применимо);
- количество строк и колонок.

## Ограничения SQL-поддержки

Реализация сделана в стиле hackathon-ready (практичность > полнота):
- нет полноценного enterprise SQL-парсера;
- фокус на `CREATE TABLE`/`INSERT`-дампы и простые структуры INSERT;
- сложные диалектные конструкции SQL могут не поддерживаться;
- `SELECT` без данных не анализируется.

Если формат SQL нестандартный, лучше загрузить CSV/Excel/JSON или SQLite-файл с таблицами.

## Как работает аутентификация

- До входа пользователь видит только вкладки **«Вход»** и **«Регистрация»**.
- Доступ к анализу данных, рекомендациям и AI-чату есть только у авторизованных пользователей.
- Пароли **не** хранятся в открытом виде: используется **Argon2** через `passlib`.
- Состояние входа хранится в `st.session_state`.
- Есть кнопка **«Выйти»**.

### Правила валидации пароля

- Минимум: **6 символов**.
- Максимум: **4096 байт в UTF-8**.
- Пароль не обрезается и не меняется «молча».
- При нарушении правил UI показывает понятную ошибку.

## База данных и что сохраняется

Используется **SQLite** + **SQLAlchemy ORM**.

По умолчанию файл базы:

```text
data/marketing_auditor.sqlite3
```

Можно изменить через `SQLITE_DB_PATH` или `DATABASE_URL`.

### Таблицы

1. `users`
   - `id`
   - `email` (уникальный)
   - `password_hash`
   - `created_at`

2. `analysis_sessions`
   - `id`
   - `user_id`
   - `original_filename`
   - `uploaded_at`
   - `mapping_json`
   - `metrics_info_json`
   - `summary_kpis_json`
   - `recommendations_json`
   - `campaign_context`

3. `chat_messages`
   - `id`
   - `analysis_session_id`
   - `role` (`user` / `assistant`)
   - `content`
   - `created_at`

## Демо-сценарий

1. Запустите приложение.
2. Войдите или зарегистрируйтесь.
3. Загрузите файл (например, `sample_data/demo_campaign.csv`).
4. Проверьте вкладки:
   - `Overview` (KPI, сигналы риска, короткое резюме);
   - `Data Mapping` (источник и сопоставление);
   - `Metrics & Results` (метрики, таблица, слабые/сильные объявления);
   - `Recommendations`.
5. (Опционально) Включите дашборды в sidebar и откройте вкладку `Dashboards`.
6. Перейдите в `AI Assistant` и задайте вопрос по текущим данным.
7. Откройте сохраненную сессию через sidebar или вкладку `History` и убедитесь, что контекст/чат восстановлены.
